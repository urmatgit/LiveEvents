# Добавление редактора времени для SomeEvent

## Обзор

Была добавлена возможность редактирования времени события в дополнение к дате события в Blazor-компоненте SomeEvents.

## Измененные файлы

### 1. SomeEventViewModel.cs
- Добавлено свойство `TimeSpan? EventTime` для хранения времени
- Добавлено свойство `DateTime? CombinedDateTime` для объединения даты и времени
- Добавлен метод `CombineDateAndTime()` для объединения значений даты и времени

### 2. SomeEvents.razor
- Заменено поле `MudDatePicker` для `EventDate` на два отдельных поля:
  - `MudDatePicker` для выбора даты
  - `MudTimePicker` для выбора времени
- Обновлены привязки данных для использования новых свойств

### 3. SomeEvents.razor.cs
- Обновлена логика обработки данных при создании/обновлении для корректной передачи даты и времени
- Добавлена логика преобразования данных при получении из API для разделения даты и времени

## Функциональность

### Новые элементы интерфейса
- Поле выбора даты (`MudDatePicker`) для `Event Date`
- Поле выбора времени (`MudTimePicker`) для `Event Time`
- Оба поля работают вместе для формирования полного значения даты и времени

### Логика обработки
- При редактировании дату и время можно выбирать отдельно
- При сохранении данные объединяются в одно значение DateTime
- При загрузке данных они автоматически разделяются на дату и время для удобного редактирования

## Технические детали

### Модель данных
- `CombinedDateTime` используется как основное свойство для синхронизации даты и времени
- `EventDate` и `EventTime` хранят отдельные компоненты даты и времени
- Метод `CombineDateAndTime()` используется для объединения значений перед отправкой в API

### Совместимость
- Изменения полностью совместимы с существующим API
- Обратная совместимость с существующими данными сохранена
- Структура базы данных остается без изменений (DateTime поле)
