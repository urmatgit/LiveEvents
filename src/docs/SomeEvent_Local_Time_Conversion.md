# Конвертация времени в локальное время для SomeEvent

## Обзор

Была реализована конвертация времени события из UTC в локальное время пользователя для корректного отображения и редактирования.

## Измененные файлы

### 1. SomeEventViewModel.cs
- Заменен импорт с `FSH.Starter.Blazor.Infrastructure.Api` на `FSH.Starter.WebApi.Catalog.Application.SomeEvents.Update.v1`
- Добавлена логика конвертации времени из UTC в локальное и обратно:
  - При получении данных из API: конвертация из UTC в локальное время
  - При сохранении данных: конвертация из локального времени в UTC
- Использован `TimeZoneInfo.ConvertTimeFromUtc()` и `TimeZoneInfo.ConvertTimeToUtc()`

## Функциональность

### Конвертация времени
- При отображении даты и времени события в интерфейсе они автоматически конвертируются в локальное время пользователя
- При сохранении данных они конвертируются обратно в UTC для хранения в базе данных
- Обеспечивается корректное отображение времени с учетом часового пояса пользователя

### Технические детали
- Все операции с временем происходят в свойстве `CombinedDateTime`
- Время хранится в UTC в базе данных (как и раньше)
- При редактировании пользователь видит время в своем локальном часовом поясе
- После сохранения время конвертируется обратно в UTC для хранения

## Преимущества

- Пользователи видят время событий в их локальном часовом поясе
- Сохраняется совместимость с существующей системой хранения (время по-прежнему хранится в UTC)
- Автоматическая конвертация без необходимости ручного вмешательства
- Корректная работа с временем в разных часовых поясах
