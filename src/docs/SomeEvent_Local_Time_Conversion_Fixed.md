# Исправление конвертации времени в локальное время для SomeEvent

## Обзор

Была исправлена ошибка конвертации времени в локальное время для Blazor WebAssembly приложения.

## Проблема

При использовании `TimeZoneInfo.ConvertTimeFromUtc()` в Blazor WebAssembly возникала ошибка:
```
The conversion could not be completed because the supplied DateTime did not have the Kind property set correctly.
```

## Решение

Использованы стандартные методы .NET для конвертации времени:
- `DateTime.SpecifyKind(dateTime, DateTimeKind.Utc).ToLocalTime()` для конвертации из UTC в локальное время
- `dateTime.ToUniversalTime()` для конвертации из локального времени в UTC

## Измененные файлы

### 1. SomeEventViewModel.cs
- Заменены вызовы `TimeZoneInfo.ConvertTimeFromUtc()` и `TimeZoneInfo.ConvertTimeToUtc()` на стандартные методы .NET
- Использован `DateTime.SpecifyKind().ToLocalTime()` для конвертации из UTC в локальное время
- Использован `ToUniversalTime()` для конвертации из локального времени в UTC
- Обновлен импорт на правильный для наследования от `UpdateSomeEventCommand`

## Функциональность

### Конвертация времени
- При отображении даты и времени события в интерфейсе они автоматически конвертируются в локальное время пользователя
- При сохранении данных они конвертируются обратно в UTC для хранения в базе данных
- Исправлена ошибка, возникавшая в Blazor WebAssembly среде

### Технические детали
- Все операции с временем происходят в свойстве `CombinedDateTime`
- Время хранится в UTC в базе данных (как и раньше)
- При редактировании пользователь видит время в своем локальном часовом поясе
- После сохранения время конвертируется обратно в UTC для хранения

## Преимущества

- Пользователи видят время событий в их локальном часовом поясе
- Исправлена ошибка, возникавшая в Blazor WebAssembly
- Сохраняется совместимость с существующей системой хранения (время по-прежнему хранится в UTC)
- Автоматическая конвертация без необходимости ручного вмешательства
- Корректная работа с временем в разных часовых поясах
